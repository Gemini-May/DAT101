    5. [In nested "for" loops, the relationship between rows and columns is defined by the order in which the loops are nested, which directly affects how elements in a grid or matrix are accessed. How this works is that the outer loop iterates over rows and the inner loop iterates over columns within each row. Changing the order generally does not affect functionality if each element is processed individually without dependency on the processing order. However, if processing order matters (for example, if results depend on accessing items row-by-row), you should keep rows as the outer loop and columns as the inner loop.]
